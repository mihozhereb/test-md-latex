## Лабораторная работа №2  
Численные методы решения нелинейных уравнений и систем

### Цель работы

- **Для нелинейных уравнений**: реализовать и сравнить численные методы поиска корней:  
  метод половинного деления, метод хорд (фиксированный конец), метод Ньютона, метод секущих и метод простой итерации.
- **Для систем двух нелинейных уравнений**: реализовать метод Ньютона и метод простой итерации, сравнить их сходимость и точность.

## Постановка задачи

### Нелинейные уравнения

- **Реализация методов**
  - **Все численные методы** (см. таблицу ниже) реализуются как отдельные функции/подпрограммы.
  - Для каждого метода должно быть видно:
    - найденный корень;
    - значение функции в корне;
    - число итераций.

- **Выбор уравнения**
  - Пользователь выбирает уравнение из зашитого набора (несколько алгебраических и трансцендентных функций).

- **Ввод исходных данных**
  - **Интервал $[a,b]$** или **начальное приближение** и **точность $\varepsilon$**.
  - Ввод возможен:
    - с клавиатуры;
    - из файла.

- **Верификация данных**
  - Проверяется наличие корня на интервале (анализ знака $f(a)f(b)$, изменения знака на сетке и т.п.).
  - Если на интервале **нет корня** или **несколько корней** – выводится информативное сообщение, метод не запускается.
  - Программа должна устойчиво реагировать на некорректный ввод (ошибки формата, пустые строки и пр.).

- **Выбор начального приближения**
  - Для методов Ньютона, секущих, хорд (фиксированный конец), простой итерации начальное приближение выбирается **автоматически** (на основе знака $f$ и $f''$, либо минимума $|f|$ на интервале).

- **Метод простой итерации**
  - Строится итерационный процесс $x_{k+1} = \varphi(x_k)$.
  - В программе проверяется **достаточное условие сходимости**:

$$
q = \max_{x\in[a,b]} |\varphi'(x)| < 1.
$$

  - Если условие не выполняется, метод не запускается и выводится сообщение:
    «Достаточное условие сходимости не выполнено.»

- **Вывод результатов**
  - По выбору пользователя:
    - **на экран**;
    - **в файл** (имя задаётся пользователем).
  - Обязательно выводятся:
    - найденный корень;
    - значение $f(x)$ в корне;
    - число итераций;
    - для метода простой итерации – параметр сходимости $q$ и $\varphi'(x_1)$ в найденном корне.

- **График**
  - Для выбранного уравнения строится график $y=f(x)$ на всём исследуемом интервале.

### Системы нелинейных уравнений

- **Выбор системы**
  - Пользователь выбирает одну из заранее заданных систем двух нелинейных уравнений.

- **Методы**
  - Метод Ньютона (6).
  - Метод простой итерации (7) с заданной $\boldsymbol{\varphi}(x,y)$ и её якобианом.

- **Ввод данных**
  - Начальные приближения $x_0, y_0$ и точность $\varepsilon$ вводятся с клавиатуры или через тестовый ввод.

- **Проверка сходимости (метод простой итерации)**
  - Проверяется достаточное условие $\|\Phi'(x,y)\| < 1$ в некоторой окрестности начального приближения.
  - При нарушении условия выводится сообщение:
    «Достаточное условие сходимости не выполнено.»

- **Вывод результатов**
  - Вектор решения $(x, y)$.
  - Значения функций $f_1(x,y), f_2(x,y)$.
  - Число итераций.
  - Вектор погрешностей / приращений по последней итерации.

- **График**
  - Отрисовываются линии уровня $f_1(x,y)=0$ и $f_2(x,y)=0$ (их пересечение – искомое решение).

## Типы ошибок во 2‑й лабораторной

### Error 1 — ошибки ввода и выбора

**Смысл:** пользователь выбрал несуществующий пункт меню или ввёл некорректный интервал.

Сообщения:

- `Некорректный номер уравнения.`
- `Некорректный номер системы.`
- `Некорректный метод.`
- `Некорректный выбор.`
- `Некорректный интервал.`

Когда возникают:

- номер уравнения / системы / метода не попал в допустимый диапазон;
- левая граница интервала не меньше правой.

### Error 2 — ошибки, связанные с корнями на интервале

**Смысл:** интервал выбран так, что на нём нет одного изолированного корня.

Сообщения:

- `На интервале корней нет.`
- `На интервале несколько корней. Уточните интервал.`

Когда возникают:

- анализ знака $f(a)f(b)$ и/или количества смен знака на сетке показывает отсутствие корня;
- либо на интервале более одного корня и метод одномерного поиска применять некорректно.

### Error 3 — нарушение достаточного условии сходимости

**Смысл:** для метода простой итерации (уравнения или системы) не выполняется условие сходимости ($\max|\varphi'|<1$ или $\|\Phi'(x,y)\|<1$).

Сообщение:

- `Достаточное условие сходимости не выполнено.`

Когда возникает:

- в функции `simple_iteration` для уравнения;
- в функции `simple_iteration_system` для системы.

Автотесты проверяют наличие именно этой подстроки.

### Error 4 — вырожденный Якобиан (системы, метод Ньютона)

**Смысл:** в методе Ньютона для системы двух уравнений матрица Якобиана стала вырожденной (определитель = 0), шаг метода посчитать нельзя.

Сообщение:

- `Якобиан вырожден.`

Когда возникает:

- внутри `newton_system`, при попытке обратить Якобиан.

### Error 5 — внутренние числовые ошибки метода

Эти ситуации выбрасываются через `ValueError` и в нормальной работе не должны доходить до пользователя (иначе это уже ошибка реализации или слишком «плохой» старт):

- `Невозможно выбрать параметр итерации.` — $\max_{[a,b]} |f'(x)| = 0$ в методе простой итерации.
- `Производная равна нулю.` — в методе Ньютона для уравнения при $f'(x)=0$.
- `Деление на ноль в методе секущих.` — при совпадающих значениях $f(x)$ в методе секущих.
- `Не удалось выбрать начальное приближение.` — не найдено удовлетворительное стартовое значение.
- `Для выбранной системы метод простой итерации не задан.` — в `simple_iteration_system`, если у системы нет $\Phi(x,y)$ и её якобиана.
- `Неизвестный метод.` — в `solve_equation` / `solve_system`, если передан неподдерживаемый номер метода.

В отчёте такие случаи можно описывать как **внутренние числовые ошибки**, которые должны быть устранены подбором интервала, начальных приближений или исправлением кода.
